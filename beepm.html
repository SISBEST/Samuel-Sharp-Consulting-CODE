BEEP! Messages

by: Samuel Sharp

A new coded messages app from samuelsharp.com!
Click here to download and play!

1. Why I Made This Game

I made this game so that kids could communicate in secret without parents getting in. It's pretty simple for the moment, but I will add new features. I I made this game instead of any other game because it was the first (well, second) idea I had. The first one was a Choose Your Own Adventure, but it would have taken me weeks to design the graphics.

2. What I Learned

My usability testing revealed that most people thought of the long beep (V 1.0) as ear-piercing and too long. So I updated that! I also added the SOS Morse because one of my testers said they would enjoy more morse code features

3. Architecture of My Game

There are 3 classes: BEEPMessages, alphanumeric, and morse. BEEPMessages is the main class. It creates the main menu. It runs 1 of the the 2 other classes based on which button the user presses.

4. Technical Implementation

The alphanumeric class encodes and decodes messages. It checks for each letter of the alphabet and places the corresponding letter in a variable that is printed into a txt file, or vice versa.

5. Most Challenging Part

The most difficult part was the decode method. The char array would not work as it did in encode, because some numbers have 2+ digits. I overcame this problem by switching to a string-based system.

6. Future Fixes and Updates

I would like to add spaces and punctuation to the encoding and decoding methods.
